# Hackathon Applications Design

Status: Draft pilot spec.

## Overview

This feature should prove the Reforge loop across profile reuse, application submission, API/server behavior, persistence, and organizer visibility.

## Design questions

- What is the canonical application lifecycle?
- Which current fields are reusable profile data vs hackathon-specific answers?
- Which existing DB tables are source-of-truth?
- What application behavior must be characterized before replacement?
- What API/package contracts must remain compatible for current consumers?

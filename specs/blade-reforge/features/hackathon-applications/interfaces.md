# Hackathon Applications Interfaces

Status: Draft pilot spec.

## Boundaries to define

- Application submission API contract
- Application read/review API contract
- Profile data consumed by application flow
- Persistence contract for application records
- Error/failure classes for closed window, duplicate application, incomplete profile, unauthenticated user

- Which apps consume current application APIs?
- Which shared constants/statuses are public contracts?

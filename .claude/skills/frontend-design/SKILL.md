# Frontend Design

Follow `docs/agentic-development/frontend-design-skill.md` from the repository root. For Blade UI work, also read `apps/blade/DESIGN_SYSTEM.md` before changing layouts, colors, cards, forms, dashboards, navigation, icons, animation, or profile/upload UI.

Before building, identify the page's job, audience, existing design-system primitives, and one intentional visual choice specific to the product. During implementation, use Forge UI tokens/components, avoid generic template layouts, keep cards visibly raised when they are cards, follow Blade's documented top-level/nested surface hierarchy, and verify desktop/mobile screenshots for meaningful visual changes.
